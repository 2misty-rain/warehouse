# 库存管理系统 - 修复说明和启动指南

## 📋 问题修复总结

### 1. ✅ 数据库模型修复
**问题**: SQLAlchemy模型定义冲突和字段类型问题
**修复**:
- 统一`Base`定义在`database.py`中,避免重复导入
- 修复`DateTime`字段,移除`timezone=True`(MySQL不支持)
- 将`reminder_type`从`Enum`改为`String`,避免枚举类型映射问题
- 添加`autoincrement=True`到主键

### 2. ✅ AI助手API修复  
**问题**: `InvalidParameter`错误
**修复**:
- 移除了dashscope不支持的`tool_choice="auto"`参数
- 简化了AI调用,先使用普通对话模式(不使用tools)
- 简化了`build_system_prompt`,避免过长提示词导致的问题

### 3. ✅ Schema修复
**问题**: 日期时间字段可能为null导致序列化失败
**修复**:
- 将`created_at`和`updated_at`设置为`Optional[datetime]`
- 移除了复杂的枚举类型验证,改用可选字符串

### 4. ✅ 错误处理增强
**问题**: 500错误没有详细信息
**修复**:
- 在AI相关接口添加了详细的try-except和日志记录
- 添加了`traceback`导入用于调试

## 🚀 系统启动指南

### 后端启动
```bash
cd f:\库存管理系统\backend
python -m uvicorn main:app --host 0.0.0.0 --port 8000
```

后端服务将在 `http://localhost:8000` 启动

### 前端启动
```bash
cd f:\库存管理系统\frontend
npm run dev
```

前端将在 `http://localhost:3000` 启动,并自动代理API请求到后端

## 🎯 AI功能说明

当前AI助手支持:
- **自然语言对话**: 可以用中文与AI交流,询问库存相关问题
- **智能回答**: AI会基于系统提示词回答关于设备库存、设备状态、设备分配等问题
- **操作记录**: 所有AI对话都会记录到`ai_logs`表

注意: 当前版本简化了工具调用功能,专注于稳定的对话交互。

## 🔧 故障排查

如果遇到问题:
1. 检查数据库连接: `python test_db.py`
2. 检查AI API密钥: 确认`.env`文件中有正确的`DASHSCOPE_API_KEY`
3. 查看后端日志: 启动时添加`--reload`参数可以自动重载代码更改
