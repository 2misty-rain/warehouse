# AI助手工具调用功能说明

## 🎯 功能概述

AI助手现在具备**安全的工具调用能力**，可以执行库存管理的增删改查操作。

### 工作流程

```
用户输入自然语言指令
    ↓
AI分析意图并生成操作指令
    ↓
前端显示确认对话框
    ↓
用户确认执行
    ↓
后端安全执行操作
    ↓
数据库更新 + 返回结果
```

## 🔒 安全机制

### 1. 用户确认机制
- 所有修改操作都需要用户明确确认
- 确认对话框清晰显示将要执行的操作
- 用户可以随时取消操作

### 2. 权限控制
- 操作通过现有的API接口执行
- 继承现有的权限验证机制
- 记录操作日志（如果已集成审计系统）

### 3. 数据验证
- 所有参数经过Pydantic模型验证
- 业务逻辑验证（如设备是否存在、是否已借出等）
- 异常处理和错误提示

## 📋 支持的操作类型

| 操作类型 | 说明 | 必需参数 |
|---------|------|---------|
| `create_inventory` | 添加新设备 | serial_number, device_id |
| `update_inventory` | 更新设备信息 | device_id + 要更新的字段 |
| `delete_inventory` | 删除设备 | device_id |
| `create_borrow` | 借出设备 | device_id, borrower |
| `return_borrow` | 归还设备 | record_id 或 device_id |

## 🚀 技术实现

### 后端

1. **smart_ai.py** - AI对话引擎
   - 增强System Prompt，指导AI生成操作指令
   - 解析AI回复中的JSON格式指令
   - 返回 `action` 和 `params` 字段

2. **main.py** - API接口
   - `/ai/chat` - 智能对话（返回操作指令）
   - `/ai/execute-action` - 执行操作（需要确认）

### 前端

1. **AIAssistant.vue** - 用户界面
   - 检测AI回复中的操作指令
   - 显示确认对话框
   - 调用执行接口
   - 显示执行结果

---

## ⚠️ 注意事项

1. **AI可能不总是生成正确的操作指令**
   - 如果AI没有返回action，只会显示文本回复
   - 用户可以继续对话，直到AI理解意图

2. **复杂操作可能需要多轮对话**
   - 例如："添加设备" → AI询问详细信息 → 用户提供 → 执行

3. **建议先用查询类问题测试AI的理解能力**
   - "当前有哪些设备？"
   - "设备CAA241100006的信息是什么？"

4. **重要操作建议手动执行**
   - 批量删除
   - 关键数据修改
   - 不确定后果的操作

## 🔧 故障排查

### 问题1：AI没有返回操作指令
**原因：** AI可能没有理解您的意图
**解决：** 
- 使用更明确的表达
- 提供完整的参数
- 尝试重新描述需求

### 问题2：操作执行失败
**原因：** 参数错误或业务逻辑冲突
**解决：**
- 查看详细错误信息
- 检查设备是否存在
- 确认状态是否正确（如借出前设备必须可用）

### 问题3：确认对话框不显示
**原因：** 前端代码问题或网络错误
**解决：**
- 检查浏览器控制台错误
- 确认后端服务正常运行
- 刷新页面重试
