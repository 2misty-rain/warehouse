基于FastAPI+Vue+MySQL的AI库存管理系统开发需求文档
项目概述
开发一个基于FastAPI后端、Vue前端和MySQL数据库的智能库存管理系统。系统具备AI自然语言处理功能，能够通过对话方式执行库存管理操作，如设备出库、状态变更、统计分析等。
技术架构
后端: FastAPI + Python 3.11+
前端: Vue 3 + Element Plus
数据库: MySQL 8.0+
AI集成: 大语言模型（通过API调用，如OpenAI GPT或其他模型）
部署: 本地部署
数据库设计
主表: inventory（库存主表）
sql

编辑



CREATE TABLE inventory (
    id INT AUTO_INCREMENT PRIMARY KEY,
    serial_number VARCHAR(50) UNIQUE NOT NULL COMMENT '序号',
    version VARCHAR(50) COMMENT '版本',
    device_id VARCHAR(50) UNIQUE NOT NULL COMMENT '设备号',
    type VARCHAR(100) COMMENT '类型',
    packaging VARCHAR(100) COMMENT '产品包装',
    device_attribute VARCHAR(100) COMMENT '设备属性',
    owner VARCHAR(100) COMMENT '归属人/责任人/甲方',
    borrower VARCHAR(100) COMMENT '领用人',
    sales_person VARCHAR(100) COMMENT '销售',
    remarks TEXT COMMENT '备注',
    supplementary_info TEXT COMMENT '补充信息记录',
    delivery_date DATE COMMENT '交付/安装时间',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
提醒表: reminders（提醒表）
sql

编辑



CREATE TABLE reminders (
    id INT AUTO_INCREMENT PRIMARY KEY,
    device_id VARCHAR(50) NOT NULL COMMENT '设备号',
    reminder_type ENUM('trial_period', 'iot_card', 'other') NOT NULL COMMENT '提醒类型',
    due_date DATE NOT NULL COMMENT '到期日期',
    description TEXT COMMENT '描述',
    is_processed BOOLEAN DEFAULT FALSE COMMENT '是否已处理',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

编辑



CREATE TABLE ai_logs (

    user_input TEXT NOT NULL COMMENT '用户输入',
    ai_parsed_action VARCHAR(200) COMMENT 'AI解析的操作',
    affected_records JSON COMMENT '受影响的记录',

1. 基础库存管理功能

批量导入/导出设备信息

Excel格式数据展示和编辑

提醒状态管理（已处理/未处理）
4. 数据可视化
设备统计图表（柱状图、饼图等）
销售趋势分析
设备状态分布
5. 数据安全与备份
AI操作日志记录
数据库定期备份
操作历史追踪
AI功能实现方案
AI集成方式
使用大语言模型API（如OpenAI GPT-4等）
通过FastAPI后端集成AI服务
实现自然语言解析和数据库操作映射
AI功能实现流程
用户在前端输入自然语言指令
前端将指令发送到后端AI接口
后端调用大语言模型API解析指令
AI解析后返回结构化操作命令
系统执行数据库操作前显示确认界面
用户确认后执行操作并记录日志
返回操作结果给前端显示

"将以下设备标记为陈育南售往银丰：CAA241100006, CAA241100007, CAA241100008"
"统计今年销售了多少台设备"
"设备CAA241100458借给A机构试用3个月"

API接口设计
设备管理接口
GET /api/devices - 获取设备列表
POST /api/devices - 添加设备
PUT /api/devices/{id} - 更新设备
DELETE /api/devices/{id} - 删除设备
POST /api/devices/batch-update - 批量更新设备
AI接口
POST /api/ai/process - 处理自然语言指令
POST /api/ai/confirm - 确认AI操作
提醒接口
GET /api/reminders - 获取提醒列表
PUT /api/reminders/{id}/process - 标记提醒为已处理
